1. APP IDENTITY
================================================================================

Name:           STL Pro Viewer
Short Name:     STL Pro
Version:        0.0.0 (react-vite-tailwind)
Type:           Browser-based 3D STL file viewer (Single Page Application)
Description:    High-performance browser-based 3D STL file viewer with mesh
                analysis, volume & surface area calculation, real-time
                rendering, and screenshot capture. All processing happens
                locally in the browser — no server uploads needed.

Privacy-first:  All STL file processing is performed entirely on-device. Files
                are never transmitted to any server or third-party service.

3. FEATURES
================================================================================

Core:
  - Load and render STL (stereolithography) 3D model files
  - Supports both binary and ASCII STL formats via Three.js STLLoader
  - Real-time 3D viewport with GPU-accelerated rendering (WebGL)
  - Perspective and Orthographic camera modes (toggleable)

3D Viewport Controls:
  - Orbit controls with damping for smooth rotation
  - Pan and zoom with mouse/touch
  - Infinite grid overlay (toggleable)
  - Axis gizmo (XYZ orientation indicator, toggleable)
  - Auto-fit bounds on model load (Drei Bounds)
  - Stage environment lighting ("city" environment map)

Display Settings (toggleable):
  - Wireframe mode
  - Edge overlay (Drei Edges component)
  - Show/hide grid
  - Show/hide axis gizmo
  - Perspective/Orthographic camera switch

Model Analysis:
  - File size (human-readable format)
  - Triangle count
  - Vertex count
  - Bounding box dimensions (X x Y x Z in mm)
  - Volume calculation (signed tetrahedron method, displayed in cm³)
  - Surface area calculation (cross-product method, displayed in cm²)

File Management:
  - Upload STL files via file picker (accepts .stl only)
  - Recent files list (up to 10 entries)
  - Files stored in IndexedDB (database: "stl-pro-files", store: "files")
  - Recent file metadata stored in localStorage (key: "recent_files")
  - Load recent files from IndexedDB storage
  - Clear individual recent files
  - Clear all recent files (removes from IndexedDB and localStorage)

Export & Sharing:
  - Screenshot capture (downloads as stl-screenshot-{timestamp}.png)
  - Native Web Share API integration (share model preview image)
  - Share includes model name and preview PNG

UI:
  - Responsive sidebar (drawer on mobile, fixed on desktop)
  - Model info overlay panel (file size, triangles, vertices, dimensions,
    volume, surface area)
  - Loading spinner with "Parsing STL Data..." overlay
  - Error toast with dismiss action
  - Model name and triangle count display in header
  - "GPU Accelerated Rendering Active" status indicator
  - Footer with Privacy Policy and Terms of Service links

PWA Support:
  - Web app manifest (manifest.json)
  - Apple touch icon for iOS home screen
  - Multiple icon sizes (192, 512, maskable variants)
  - Favicon (ICO format)
  - Theme color: #1a1a2e

7. DATA STORAGE
================================================================================

IndexedDB:
  - Database name: "stl-pro-files"
  - Object store: "files"
  - Key: filename (string)
  - Value: ArrayBuffer (raw STL binary data)
  - Purpose: Persists STL file data across sessions for "Recent Files"
  - Data never leaves the device

localStorage:
  - Key: "recent_files"
  - Value: JSON array of ModelStats objects (max 10 entries)
  - Fields per entry: name, size, triangles, vertices, dimensions, volume,
    area, lastOpened
  - Purpose: Quick access to recent file metadata without loading full STL data
  - Data never leaves the device

In-Memory:
  - THREE.BufferGeometry — loaded STL geometry held in component state
  - Display settings — held in useState within Home component
  - Geometry is discarded when a new file is loaded or the model is cleared

8. STL PROCESSING
================================================================================

Loading Pipeline:
  1. User selects .stl file via <input type="file" accept=".stl">
  2. File read as ArrayBuffer via file.arrayBuffer()
  3. ArrayBuffer saved to IndexedDB for persistence
  4. STLLoader.parse(arrayBuffer) converts to THREE.BufferGeometry
  5. Bounding box computed (geom.computeBoundingBox())
  6. Vertex normals computed (geom.computeVertexNormals())
  7. Model statistics calculated (triangles, vertices, dimensions, volume, area)
  8. Geometry rendered in Three.js Canvas

STL Format Support:
  - Binary STL (most common, compact)
  - ASCII STL (text-based, larger file sizes)
  - Both parsed by Three.js STLLoader

Volume Calculation:
  - Method: Signed tetrahedron volume
  - Formula: volume += p1.dot(p2.cross(p3)) / 6.0 for each triangle
  - Applied to all triangles in non-indexed geometry
  - Returns absolute value of total signed volume
  - Displayed in cm³ (divided by 1000)

Surface Area Calculation:
  - Method: Cross-product triangle area
  - Formula: area += |v1 × v2| * 0.5 for each triangle
  - v1 = p2 - p1, v2 = p3 - p1
  - Displayed in cm² (divided by 100)

Triangle Count:
  - Computed as: geom.attributes.position.count / 3
  - Each triangle has 3 vertices

Vertex Count:
  - Computed as: geom.attributes.position.count

Dimensions:
  - Extracted from bounding box: bbox.getSize(dimensions)
  - Reported as X x Y x Z in mm
